## 1. Команды администратора

| Команда | Что делает |
|---|---|
| `/admin` | Главная панель: текущая неделя, статус записи (открыта/закрыта), число подписчиков, вместимость, сколько записано / в ожидании / отказались / отменили, и список команд. |
| `/capacity` | Показывает текущую вместимость (максимум участников). |
| `/setcapacity 25` | Меняет вместимость на 25 **прямо сейчас** (без перезапуска). Если увеличить — всем из листа ожидания придёт уведомление о свободных местах. |
| `/broadcast Ваш текст` | Разослать текстовое сообщение всем подписчикам. |
| *(отправить фото)* | Отправьте боту фото (можно с подписью) — он предложит кнопку **«📩 Разослать всем»**. Нажмите — фото уйдёт всем подписчикам. |
| `/texts` | Показывает список всех текстов, которые можно менять. |
| `/settext welcome Новый текст` | Меняет текст `welcome` на «Новый текст». |
| `/resettext welcome` | Возвращает текст `welcome` к стандартному. |
| `/trigger_open` | Вручную запускает понедельничную рассылку (разослать карточку записи всем). Удобно для теста или вне расписания. |
| `/trigger_close` | **Закрывает запись**: новые «Да, участвую» и «Занять место» больше не принимаются. Отменить своё участие по-прежнему можно. |
| `/trigger_reminder` | Вручную рассылает пятничные напоминания (только записавшимся). |

**Команды для всех (и для обычных пользователей):**

| Команда | Что делает |
|---|---|
| `/start` | Начать пользоваться ботом / подписаться. |
| `/stop` | Отписаться от рассылки. |
| `/help` | Короткая справка. |

## 2. Все тексты, которые можно менять

Команда `/texts` покажет список «ключей». Команда `/settext <ключ> <новый текст>` меняет текст, `/resettext <ключ>` — возвращает стандартный.

**Про подстановку `{event_info}`.** В некоторых текстах можно написать `{event_info}` — бот сам подставит туда описание времени встречи (настраивается в `.env`, параметр `EVENT_INFO`; по умолчанию «каждое воскресенье в 18:30»). Это работает в текстах: `welcome`, `monday_open`, `decline_no`. Другие фигурные скобки `{...}` лучше не использовать.

| Ключ | Где появляется |
|---|---|
| `welcome` | Приветствие при команде `/start` (до подписки). Можно использовать `{event_info}`. |
| `subscribed` | Сообщение после нажатия «Подписаться» (поздравление с регистрацией). |
| `unsubscribed` | После команды `/stop` (отписка). |
| `monday_open` | Главная понедельничная рассылка-карточка с кнопками «Да/Нет». Можно использовать `{event_info}`. |
| `confirm_registered` | После «Да, участвую», когда место есть. |
| `confirm_waiting` | После «Да, участвую», когда мест нет (попал в лист ожидания). |
| `decline_no` | После «Нет, не смогу». Можно использовать `{event_info}`. |
| `cancelled` | После отмены регистрации (человек был записан). Место освобождается. |
| `cancelled_waiting` | После выхода из листа ожидания. |
| `spot_freed` | Уведомление тем, кто в листе ожидания, когда освободилось место (с кнопкой «Занять место»). |
| `spot_taken` | Всплывающее сообщение, если место уже успели занять. |
| `spot_taken_success` | После нажатия «Занять место» — место ваше. |
| `reminder` | Пятничное напоминание записавшимся. |
| `already_registered` | Всплывающее сообщение, если нажать «Да», когда уже записан. |
| `registration_closed` | Всплывающее сообщение, если запись уже закрыта (`/trigger_close`). |

**Примеры:**

```
/settext reminder ⏰ Завтра беговой клуб! Не забудьте кроссовки 😊
/settext monday_open 🏃 Открыта запись на клуб!

Когда: {event_info}.

Жми «Да»!
/resettext reminder
```

> Тексты, рассылки и вместимость меняются командами **сразу и без перезагрузки**. Время рассылок, часовой пояс и список админов — только через файл `.env` + перезапуск (см. ниже).

## 3. Как поменять время еженедельных рассылок

Откройте файл `.env` и поправьте нужные строки:

```env
MONDAY_OPEN_TIME=10:00
FRIDAY_REMINDER_TIME=09:00
TIMEZONE=Europe/Moscow
```

- **`MONDAY_OPEN_TIME`** — во сколько в понедельник рассылается карточка записи.
- **`FRIDAY_REMINDER_TIME`** — во сколько в пятницу уходят напоминания.
- **`TIMEZONE`** — часовой пояс (название зоны по списку IANA, например `Europe/Moscow`, `Europe/Kaliningrad`, `Asia/Yekaterinburg`).

Время — в формате `ЧЧ:ММ`, 24 часа (например `18:30`, `09:00`).

**После изменения `.env` обязательно перезапустите бота** (изменения расписания вступают в силу только после перезапуска). Если вы не знаете, как перезапустить — попросите того, кто устанавливал бота, или используйте панель хостинга.

## 4. Другие настройки в файле `.env`

| Параметр | Что это |
|---|---|
| `BOT_TOKEN` | Токен бота (выдаёт [@BotFather](https://t.me/BotFather)). Менять не нужно, если только не пересоздаёте бота. |
| `ADMIN_IDS` | Список ID администраторов через запятую, например `ADMIN_IDS=123456789,987654321`. |
| `CAPACITY` | Стандартное число мест. Удобнее менять на ходу командой `/setcapacity`. |
| `EVENT_INFO` | Подставляется в тексты вместо `{event_info}` (например `каждое воскресенье в 18:30`). |
| `DATABASE_PATH` | Где хранится база данных. Менять не нужно. |

Все изменения в `.env` требуют перезапуска бота.

## 5. Кратко на неделю

- **Понедельник** — бот сам открывает запись. Нужно открыть раньше/позже — `/trigger_open`. Нужно срочно закрыть — `/trigger_close`.
- **В течение недели** — смотрите статистику `/admin`; меняйте вместимость `/setcapacity`; при необходимости делайте рассылку `/broadcast` или присылайте фото.
- **Пятница** — бот сам шлёт напоминания. Нужно вручную — `/trigger_reminder`.
- **Перед забегом** — проверьте `/admin`, сколько человек записано.
